## Design Principles

- Every runtime behavior should be represented by explicit DSL directives.
- Avoid implicit path/model-based behavior in runtime execution.
- Keep transformations opt-in via directives.

## Typical Directive Categories

- Request mapping: `req_map ...;`
- Response mapping: `resp_map ...;`
- SSE parsing/transform: `sse_parse ...;`

## Authoring Rules

- Nginx-style syntax
- Every statement ends with `;`
- Validate configs before running in production
